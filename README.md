# Probabilistic Spelling Correction System

A full-stack spelling corrector combining **SymSpell**, **BERT (bert-base-uncased)**,
**Levenshtein edit distance**, and a **bigram language model** — built on a
medical science corpus (~100 000+ word tokens).

---

## Project Structure

```
spell_system/
├── backend/
│   ├── app.py              ← Flask REST API server
│   ├── nlp_engine.py       ← Core NLP: SymSpell + BERT + bigram + Levenshtein
│   ├── corpus_builder.py   ← Downloads medical corpus, builds frequency dict + bigrams
│   ├── requirements.txt    ← Python dependencies
│   └── data/               ← Generated by corpus_builder.py (auto-created)
│       ├── corpus.txt
│       ├── frequency_dict.txt
│       └── bigrams.json
│
└── frontend/
    ├── index.html          ← Main UI
    ├── style.css           ← Stylesheet
    └── app.js              ← API calls + rendering
```

## Setup

### 1. Prerequisites

- Python 3.9 or higher
- pip
- A modern browser (Chrome, Firefox, Edge, Safari)
- Internet connection for first run (to download BERT model weights ~440 MB)

---

### 2. Install Python dependencies

```bash
cd backend
pip install -r requirements.txt
```

---

### 3. Build the corpus (run once)

This downloads four public-domain medical texts from Project Gutenberg, cleans
them, and builds the frequency dictionary and bigram model:

```bash
python corpus_builder.py
```

Expected output:
```
=== Corpus Builder ===
  Downloading https://www.gutenberg.org/cache/epub/40726/pg40726.txt ...
  Downloading https://www.gutenberg.org/cache/epub/4483/pg4483.txt ...
  ...
  Total tokens: 421,832
  Saved corpus.txt
  Saved frequency_dict.txt  (18,204 unique words)
  Saved bigrams.json  (94,312 bigrams)
=== Build complete. You can now start the server. ===
```

---

### 4. Start the Flask server

```bash
python app.py
```

On first start, BERT (`bert-base-uncased`) is downloaded from HuggingFace
(~440 MB, one-time download). Subsequent starts load from cache instantly.

Expected output:
```
Initialising NLP Engine ...
Loading BERT model 'bert-base-uncased' ...
BERT model loaded.
NLP Engine ready. vocab=18204, BERT=ON
 * Running on http://0.0.0.0:5000
```

---

### 5. Open the frontend

Open `frontend/index.html` in your browser.

> **Note:** Because the frontend calls `http://localhost:5000`, open the file
> directly from disk (`file:///...`) or serve it with any static server:
> ```bash
> cd frontend
> python -m http.server 8080
> # then visit http://localhost:8080
> ```

## API Reference

All endpoints are served from `http://localhost:5000`.

### `POST /api/check`
Spell-check a piece of text.

**Request body:**
```json
{ "text": "The pashent has diabetis and hypertenshon." }
```

**Response:**
```json
{
  "tokens"  : ["The", " ", "pashent", " ", "has", " ", ...],
  "errors"  : [
    {
      "token_index" : 2,
      "word"        : "pashent",
      "clean"       : "pashent",
      "type"        : "nonword",
      "candidates"  : [
        { "word": "patient", "edit_distance": 2, "frequency": 8432,
          "bert_score": -1.23, "score": 4.56 }
      ]
    }
  ],
  "stats"   : { "total": 8, "nonword": 2, "realword": 0, "correct": 6 },
  "bert_on" : true
}
```

---

### `GET /api/corpus/words`
Return vocabulary sorted by frequency.

| Param    | Type   | Default | Description                      |
|----------|--------|---------|----------------------------------|
| `q`      | string | `""`    | Filter words containing substring |
| `limit`  | int    | `200`   | Page size (max 1000)             |
| `offset` | int    | `0`     | Pagination offset                |

---

### `GET /api/corpus/bigrams`
Return top bigrams sorted by count.

| Param    | Type   | Default | Description                      |
|----------|--------|---------|----------------------------------|
| `q`      | string | `""`    | Filter pairs containing substring |
| `limit`  | int    | `100`   | Page size (max 500)              |
| `offset` | int    | `0`     | Pagination offset                |

---

### `POST /api/edit-distance`
Compute Levenshtein distance between two words.

**Request:** `{ "a": "pashent", "b": "patient" }`
**Response:** `{ "a": "pashent", "b": "patient", "distance": 2 }`

---

### `GET /api/status`
Server health check.

**Response:** `{ "ok": true, "bert_on": true, "vocab": 18204 }`

## NLP Architecture

```
Input Text
    │
    ▼
Tokeniser  ─────────────────────────────────────────────────────┐
    │                                                            │
    ▼                                                            │
Is word in vocabulary?                                           │
    │                                                            │
   NO ──► Non-word Error                                         │
    │         │                                                  │
    │         ▼                                                  │
    │     SymSpell lookup (edit distance ≤ 2)                    │
    │         │                                                  │
    │         ▼                                                  │
    │     Re-rank candidates:                                    │
    │       • log P(candidate)       [corpus frequency]          │
    │       • bigram context score   [P(c | prev_word)]          │
    │       • BERT score             [masked LM log-prob]        │
    │         │                                                  │
    │         ▼                                                  │
    │     Top-8 suggestions (with edit distance labels)          │
    │                                                            │
   YES ──► Real-word check                                       │
               │                                                 │
               ▼                                                 │
           BERT: mask word → compare log-prob vs confusables     │
           Bigram fallback: P(word|prev) vs P(alt|prev)          │
               │                                                 │
          flagged? ──YES──► Suggest confusable alternatives      │
               │                                                 │
               NO ──────────────────────────────────────────────►│
                                                                 │
                                                                 ▼
                                                           Output JSON
```

### Technique Summary

| Technique              | Role                                         |
|------------------------|----------------------------------------------|
| SymSpell               | Fast candidate generation (non-word errors)  |
| Levenshtein            | Edit distance annotation on every suggestion |
| BERT (bert-base-uncased)| Context scoring & real-word detection       |
| Bigram LM              | Fallback context + candidate re-ranking      |
| Noisy Channel Model    | Final score = log P(word) + context score    |
| Confusables Table      | Seed set for real-word error detection       |

## Troubleshooting

| Problem | Solution |
|---|---|
| `ModuleNotFoundError` | Run `pip install -r requirements.txt` |
| `frequency_dict.txt not found` | Run `python corpus_builder.py` first |
| BERT not loading | Check internet connection; first run downloads ~440 MB |
| CORS error in browser | Open frontend via `python -m http.server` not `file://` |
| Server offline message | Make sure `python app.py` is running |
